Splash Screen validation report

I ran the Splash Screen (splash.html) through the W3C Nu HTML Checker and on the first pass it flagged one thing - the meta refresh attribute was missing a space after the semicolon (I had written content="4;url=home.html" which the browser was rendering fine but the spec requires a space). After fixing it to content="4; url=home.html" the page passed with zero errors and zero warnings. The CSS spinner animation and countdown JavaScript are both correctly structured with no validation issues.

Action Impact Simulator validation report

When I first validated the AIS page (ais.html) I got a heading hierarchy error because I had jumped from <h1> straight to <h3> on the action card titles - skipping the h2 level which isn't in line with accessibility hierarchy. I fixed this by changing the card titles to <h2> and updating the matching CSS selectors so the styling still applied properly. On the second run the page passed with zero errors and zero warnings. All six action card images have proper alt text and the embedded style block contains well over the required five CSS rules.

Content Page validation report

The Content Page (content_ST2.html) actually passed on the very first attempt with zero errors, which was nice because by the time I got to it I had already learnt what to avoid from the earlier two pages. Heading hierarchy runs h1 to h2 to h3 in proper order. All content images include descriptive alt text. The internal jump navigation carries an aria-label="Page sections" so screen readers announce it as a landmark, and all anchor links inside it target valid section ids. The one small warning I got initially was about the <section> without a heading right after the hero but that was actually a text formatting thing - not real content missing a heading.

Reflection on Validation

Running the W3C validator on all three of my pages was really useful because it caught things that I wouldn't have spotted just by looking at the browser output. The meta refresh space issue on the Splash Screen is a good example - the page looked exactly right and worked exactly right, but the syntax was technically wrong according to the spec. Learning that even tiny attribute typos can fail validation was a lesson worth having early.

The heading hierarchy fix on the AIS was probably my biggest takeaway. I had jumped straight from h1 to h3 because I thought the h3 looked better sized for the card titles - but that's what CSS is for. From now on I'll always keep heading order semantic first and then style them however I want with CSS. Running the validator early and often rather than only at the end made the whole process much less painful.
