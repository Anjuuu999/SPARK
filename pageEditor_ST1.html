Home Page

Technical Description

The Home Page for the "SPARK" project was developed to introduce our chosen Sustainable Development Goal: Climate Action. I implemented the shared HTML template using semantic tags including header, nav, main, and footer tag to ensure a consistent structure across the entire site. For the core content, I used section tags to separate the hero introduction, the mission strategy area, and the theme exploration portal.

Technically, the layout relies heavily on CSS Grid and Flexbox. I used display: grid with grid-template-columns: repeat(4, 1fr) to create a responsive four-column layout for the mission cards. Each card includes a transition effect (transform: translateY(-10px)) and border-color changes on hover to provide a tactile feel. One of the most complex features I implemented is the sliding background navigation bar. This uses a span tag element within the nav tag that shifts its left position and width using CSS sibling selectors (li:hover ~ span) and transitions. This allows the black "pill" background to glide smoothly behind the text as the user hovers over different menu items.

The visual identity is defined by a dark-themed aesthetic (#121212) contrasted with "greenyellow" and "emerald" accents (#2ecc71) to represent sustainability. I used a linear-gradient overlay on the background image within the body selector to ensure that the white text remains legible regardless of the image brightness.

Accessibility

Accessibility was integrated into the Home Page through both structural and visual choices. I ensured that every image, from the Earth logo to the mission icons, contains descriptive alt text (e.g., "Icon representing solar panels and clean energy") so screen reader users understand the context of the graphics. I used semantic heading levels (h1 tag for the main call to action and h3 tag for cards) to create a logical document outline. To support keyboard-only users, I maintained a high color contrast ratio between the green accents and the dark backgrounds. Furthermore, the navigation bar was positioned using fixed CSS to ensure it is always available to the user, and I ensured that all interactive elements like the "Portal Buttons" have clearly defined hover states to indicate they are clickable.

Gallery Page

Technical Description

The Gallery Page was developed to provide an immersive, interactive visual exploration of eight key climate impact themes. I used a CSS Grid layout (repeat(4, 1fr)) to display the gallery cards in a clean, professional grid that remains organized across different screen sizes. Each card is structured as a thumbnail image with a lower title bar. I utilized HTML5 data attributes (data-title and data-info) on each card element to store descriptive content. This approach keeps the HTML clean while making it easy for JavaScript to access the relevant text for each specific card when clicked.

The interactivity of the page is powered by JavaScript DOM manipulation. I used querySelectorAll to select all gallery cards and implemented a forEach loop to attach a click event listener to each one. When a card is clicked, the script captures the image source and the text from the data attributes, then dynamically populates the Extended View (Modal) before displaying it. I also implemented a specific requirement for User Customization using two functional buttons within the modal. These buttons use classList.toggle to switch between CSS classes: alt-font (changing the text to Courier New) and alt-color (switching to a high-contrast dark theme). This allows users to adjust the viewing experience to their personal preference or accessibility needs.

Accessibility

Accessibility was a core focus during the construction of the gallery. Every thumbnail in the grid and the modal images includes meaningful alt text (e.g., "A melting glacier" or "Solar panels") to ensure visually impaired users using screen readers can understand the content. The modal is designed with a clear, high-contrast close button (×) that is easy to locate. I ensured that the gallery headers and card titles use a consistent heading hierarchy for better document navigation. Furthermore, the User Customization controls provide a functional benefit for accessibility, allowing users to switch to a monospaced font or a different color scheme to improve readability depending on their vision requirements

Content Page

Technical Description

The Content Page, titled "Water and Climate Change," was created to provide a structured and detailed resource on how SDG 13 impacts the global water cycle. I utilized the shared HTML template to maintain group branding while implementing a clean, single-column main content area (.main-content-area) to prioritize readability. The page is divided into four thematic sections-The Water Cycle, Melting Glaciers, Extreme Weather, and Warming Oceans-each marked with an h2 tag heading and a unique id attribute. I also included a custom .impact-fact box with a distinct blue border to highlight critical statistics, creating a strong visual hierarchy.

To enhance user experience on this long-form page, I developed an internal navigation menu (.page-nav) at the top, which uses Flexbox to organize a list of jump-links. This allows users to navigate directly to specific topics without excessive scrolling. Additionally, I implemented a "Go to Top" button using position: fixed, ensuring it remains in the bottom-right corner as the user scrolls. This button relies entirely on HTML and CSS anchors to function, meeting the coursework requirement for a non-JavaScript navigation shortcut. All images were styled with a consistent .content-img class, featuring border-radii and box-shadows to ensure a professional and unified visual style.

Accessibility

For the Content Page, I strictly followed a semantic heading hierarchy, starting with a single h1 tag for the page title followed by logical h2 tag sub-sections. This structure allows screen reader users to grasp the document outline easily. Each of the four images includes descriptive alt text (e.g., "Mountain and the water cycle" and "Cracked dry earth in a desert landscape") to convey meaning to visually impaired users. I prioritized readability by using a line-height of 1.8 and high-contrast text (#333) against a light background. To support keyboard navigation, I ensured that the internal navigation links and the "Go to Top" button have clear hover and focus states. Furthermore, the navigation container includes an aria-label to provide context for assistive technologies.

Challenges and Lessons Learned

One of the first major hurdles I encountered was the creation of the sliding navigation bar background. My goal was to have a black "pill" shape glide smoothly behind the links as the user hovered over them. Initially, I struggled with the positioning logic; the span would either stay static or move to the wrong location because I hadn't accounted for the varying widths of the list items like "Simulator" versus "Team." I solved this by meticulously calculating the left offset and width values for each nth-child state in my CSS. This taught me the precision required for complex CSS transitions and how to use the general sibling selector (~) to control an element based on the state of its preceding siblings.

While designing the Home Page mission cards, I initially used Flexbox. However, I found that when the text content in one card was longer than the others, the alignment of the "Key Actions" list would become uneven across the row, making the layout look messy. To fix this, I refactored the entire section using CSS Grid. By defining grid-template-columns: repeat(4, 1fr), I achieved perfect symmetry. I also learned to nest a flex container inside the grid item using flex-direction: column and margin-top: auto on the highlights section. This ensured that regardless of text length, the action lists always aligned perfectly at the bottom of the cards.

Implementing the interactive gallery modal presented a significant JavaScript debugging challenge. Initially, my script would open the modal, but it would only display the image from the first gallery card, regardless of which card was actually clicked. I realized I was not correctly targeting the specific instance of the clicked element. I solved this by utilizing HTML5 data attributes and using getAttribute within the event listener to pull the unique title and info for the clicked card. Additionally, I initially had issues with the customization toggles (font and color change) where styles would "stack" and become unreadable. I fixed this by creating dedicated .alt-font and .alt-color classes and using classList.toggle to ensure the logic was clean and easily reversible for the user.

Finally, a recurring challenge was managing the Global CSS file for the entire group. Being responsible for the shared template meant I had to write styles that wouldn't "break" the pages built by my three teammates. I encountered a "broken" image path issue when we first integrated our files because our folder structures weren't perfectly aligned. I solved this by standardizing our file naming conventions and using relative paths. This experience taught me the vital importance of writing non-conflicting, modular code and communicating clearly with a team to maintain a unified design system.

Compliance

All content on my three pages is original, written by me from my own research. Where I referenced external facts and statistics, I cited the original source in-text and included a full entry in the References section. No copyrighted articles, images, or code have been reproduced verbatim.

The images used on my pages come from either the SPARK team's own image folder or from stock sources that permit non-commercial educational use. Any external image sources are attributed in the References section. No branded logos, trademarks, or celebrity likenesses appear anywhere on my pages, which keeps the project on the safe side of Janet acceptable use guidelines.

The site collects no personal data, uses no third-party trackers, and stores nothing in cookies or local storage. All interactivity happens client-side within the browser session and disappears when the user closes the tab. This means the project sits comfortably within data protection expectations for a student educational project.

Accessibility has been treated as a compliance requirement rather than a nice-to-have. Semantic HTML tags, descriptive alt text on every image, a strict heading hierarchy, ARIA labels on non-obvious landmarks, and adequate contrast ratios all sit within the Janet expectations for public-facing educational content and are consistent with the WCAG guidance covered in the module lectures.

Nothing on the pages is offensive, defamatory, or promotes illegal activity. The tone is educational and constructive, aligned with the SDG 13 climate action theme.
